# Knowledge: Strategy Department

## Các Khung Outline

| Framework | Dùng khi | Cấu trúc |
|-----------|---------|----------|
| **Problem-Solution** | Explainer, how-to | Problem → Why it matters → Solution → Steps → Result |
| **Inverted Pyramid** | Report, news | Key finding → Details → Background → Deep dive |
| **AIDA** | Sales, persuasion | Attention → Interest → Desire → Action |
| **Story Arc** | Narrative, case study | Setup → Conflict → Rising action → Climax → Resolution |
| **Listicle+** | Social, blog | Hook → Item 1-N (each with mini-story) → Wrap |
| **Comparison** | Review, analysis | Intro → Option A → Option B → Critique → Recommendation |
| **SCQA** | Consulting, strategy | Situation → Complication → Question → Answer |

## Kỹ thuật Tạo góc tiếp cận

1. **Contrarian** — "Tất cả đều sai về [X]. Đây là sự thật"
2. **Data-first** — "[Số liệu sốc] — và đây là ý nghĩa"
3. **Personal** — "Tôi đã [trải nghiệm X] và nhận ra..."
4. **Prediction** — "[Trend X] sẽ thay đổi [Y] trong 2 năm tới"
5. **Hidden insight** — "Điều ít ai biết về [X]"
6. **Framework** — "Quy tắc 3 bước để [đạt kết quả Y]"

## Quy tắc Xây dựng Luận điểm

- 1 thesis = 1 câu khẳng định CÓ THỂ tranh luận
- **Tốt**: "Remote work hiệu quả hơn office CHỈ KHI team có async culture"
- **Xấu**: "Remote work là xu hướng" (quá hiển nhiên, không tranh luận được)
- Thesis phải testable — có thể prove/disprove bằng evidence
