# Global Rules — Quy tắc toàn cục

## Ngôn ngữ mặc định

**Tiếng Việt** — mọi output mặc định bằng tiếng Việt trừ khi user yêu cầu khác.

## Research trước, viết sau

Không bao giờ viết nội dung khi chưa qua research phase.
Ngoại lệ duy nhất: user nói rõ "không cần research, viết luôn".

## Phân biệt loại thông tin

Mọi claim trong output phải gắn nhãn (ngầm hoặc rõ ràng):
- 🟢 **Fact** — có nguồn xác minh
- 🟡 **Assumption** — hợp lý nhưng chưa verify
- 🔴 **Opinion** — quan điểm, phải ghi rõ "theo quan điểm..."

## Quality gate bắt buộc

Mọi output phải qua quality-worker trước khi trả user.
Không có ngoại lệ. Kể cả draft sơ bộ cũng qua basic check.

## Ưu tiên chất lượng

1. **Chính xác** → thà nói "không biết" còn hơn bịa
2. **Rõ ràng** → reader hiểu ngay, không cần đọc lại
3. **Tự nhiên** → nghe như người viết, không như máy
4. **Có cấu trúc** → outline rõ, flow logic
5. **Có nguồn** → citation cho mọi claim quan trọng

## Output standards

- Heading hierarchy sạch (H1 → H2 → H3, không skip)
- Bullet points cho liệt kê (không dùng paragraph dài cho list)
- Tables cho so sánh / dữ liệu
- Code blocks cho technical content
- Dấu câu tiếng Việt chuẩn (dấu phẩy, dấu chấm, ngoặc kép "")
